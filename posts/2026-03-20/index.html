March 20, 2026 dev-tools llm open-source releases

Open Source Releases

  • llm-guard-kit 0.37.0 — Llm-guard-kit offers a suite of tools for LLM safety, including a ToolCallExtractor for detecting tool usage, GuardPipeline for composing safety checks, and ModelTransferCalibrator for adapting guards across models. It also features ActiveSamplingStrategy for efficient red-teaming and ChatGuard with real-time moderation capabilities.
  • beam-infer 1.0.0 — Beam-infer provides tools for simulating beam dynamics and performing parameter inference based on raster scanning models. The package includes numerical solvers for beam transport and statistical methods for extracting model parameters from simulation data. It is aimed at researchers and engineers working on accelerator physics.
  • spatialformer 0.1.4 — Spatialformer is a single-cell foundation model designed to capture spatial cell-cell colocalization and subcellular molecular co-occurrence patterns. The model leverages transformer architectures to integrate spatial transcriptomics data, enabling downstream tasks such as cell type mapping and integration with spatial atlases.
  • penbot 2.0.0 — Penbot is a penetration testing framework specifically tailored for AI chatbots, offering automated attack vectors to evaluate robustness against prompt injection, jailbreaking, and data leakage. The framework provides a modular plugin system for adding new test cases and generates detailed reports.
  • aitta-client 0.3.3 — Aitta-client is a Python library that simplifies interaction with the Aitta HPC ML inference platform, handling job submission, data transfer, and result retrieval. It abstracts the platform’s REST API and provides asynchronous utilities for scaling experiments across GPU clusters. The 0.3.3 release adds improved retry logic and better logging.

AI Dev Tools

  • github/spec-kit — Toolkit to help developers adopt Spec-Driven Development (SDD) practices, providing templates and guidelines for writing technical specifications before implementation. It encourages clear communication and reduces ambiguity in software projects.

Today’s Synthesis

[github/spec-kit] gives you a lightweight way to nail down exactly what safety guarantees your LLM‑powered feature needs before a single line of guard code is written—think input sanitization thresholds, allowed tool call patterns, and latency budgets. With those specs in hand, you can plug [llm-guard-kit 0.37.0]’s GuardPipeline and ToolCallExtractor directly into your service, configuring each check to match the spec’s parameters and using ModelTransferCalibrator to port the same guards across models as you experiment. This spec‑first → implement → adversarial‑test loop turns vague safety wishes into a repeatable, measurable engineering process.
