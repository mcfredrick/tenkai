## Open Source Releases
- **[llm-guard-kit 0.37.0](https://pypi.org/project/llm-guard-kit/0.37.0/)** — A safety toolkit for LLMs that bundles a ToolCallExtractor, GuardPipeline, and ModelTransferCalibrator, plus ActiveSamplingStrategy for targeted red‑team runs. Handy if you want to plug guardrails into an existing inference service without reinventing the wheel. 🛡️
- **[beam-infer 1.0.0](https://pypi.org/project/beam-infer/1.0.0/)** — Provides numerical solvers for beam transport and statistical wrappers to infer model parameters from raster‑scan simulation data. Aimed at folks doing accelerator physics or similar simulations who need a quick way to go from raw scans to usable parameters. ⚙️
- **[spatialformer 0.1.4](https://pypi.org/project/spatialformer/0.1.4/)** — A transformer‑based foundation model for single‑cell spatial transcriptomics that learns cell‑cell colocalization and molecular co‑occurrence patterns. Useful for downstream tasks like cell‑type mapping or predicting spatial niches without hand‑crafting features. 🔬
- **[penbot 2.0.0](https://pypi.org/project/penbot/2.0.0/)** — A modular penetration‑testing framework for AI chatbots that automates prompt‑injection, jailbreak, and data‑leakage checks, spitting out detailed reports. If you’re shipping a LLM‑powered chat interface, this lets you run a basic security sweep before each release. 🕵️‍♂️
- **[aitta-client 0.3.3](https://pypi.org/project/aitta-client/0.3.3/)** — Python wrapper around the Aitta HPC ML inference platform that handles job submission, data transfer, and result retrieval with async utilities. Makes scaling experiments across GPU clusters feel less like wrangling a batch system and more like calling a local function. 🚀

## AI Dev Tools
- **[github/spec-kit](https://github.com/github/spec-kit)** — A collection of templates and guidelines for Spec‑Driven Development, pushing teams to write clear technical specs before code lands. Reduces the usual back‑and‑forth when requirements are vague, especially useful for cross‑functional ML projects. 📝

## Today's Synthesis

**[llm-guard-kit](https://pypi.org/project/llm-guard-kit/0.37.0/)** gives you a ready‑made guardrail stack (ToolCallExtractor, GuardPipeline, ModelTransferCalibrator, ActiveSamplingStrategy) that you can drop into any inference service, while **[penbot](https://pypi.org/project/penbot/2.0.0/)** automates prompt‑injection, jailbreak, and data‑leakage checks for LLM‑powered chatbots. By wiring penbot’s test suite into a CI step that runs before each deploy, you generate a red‑team report highlighting weak spots. Feed those findings straight into llm-guard-kit’s ActiveSamplingStrategy to focus extra validation on the exact failure modes penbot uncovered, and then use **[aitta-client](https://pypi.org/project/aitta-client/0.3.3/)** to spin up a temporary GPU cluster, run the targeted red‑team samples at scale, and collect the calibrated guardrail metrics without wrestling with batch scripts. The result is a repeatable pipeline: security sweep → adaptive guardrail tuning → scaled validation — letting you ship LLM features with confidence that the safety net has been stress‑tested on the same data that exposed the risks. 🛡️🕵️‍♂️🚀
