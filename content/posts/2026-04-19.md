## Today's Synthesis

If you are already shipping on-device inference, establish a baseline stack using concrete configurations: use JAX/Flax with `jax.jit` for portable transformer training, then compile to Octomil’s hardware-aware configs via its CLI (`octomil deploy --model <path> --optimize`). Validate that [opencode v1.1.17](https://github.com/anomalyco/opencode/releases/tag/v1.1.17) enforces executable permissions through your Docker build stage (e.g., run `RUN chmod +x ./bin/*` and confirm via `diff` against the release artifacts) and that its attachment-type detection flags mismatched binaries before deployment. Gate CI with: JAX unit tests, a lightweight Octomil benchmark pass (`octomil benchmark --threshold 0.95`), and a file-permission audit using the OpenCode release diff. If your pipeline relies on JIT for latency, the incremental cost of adding Octomil’s optimizer and a permission sanity check is low relative to the risk of silent failures in prod.
