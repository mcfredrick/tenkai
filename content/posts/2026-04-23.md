## Model Releases 📖

- **[unsloth/Qwen3.6-27B-GGUF Quantized Release](https://huggingface.co/unsloth/Qwen3.6-27B-GGUF)** — Unsloth drops GGUF quantized versions of the Qwen3.6-27B dense model for those running inference locally. It uses imatrix quantization and plays nice with standard transformer serving endpoints, so you can keep the library on your laptop. 🤖
- **[openai/privacy-filter Token Classification Model](https://huggingface.co/openai/privacy-filter)** — OpenAI releases a token classification model designed to filter sensitive data from text streams. It ships in transformers, ONNX, and safetensors formats with Apache 2.0 licensing, making it easy to slot into privacy pipelines without legal drama. 📄
- **[moonshotai/Kimi-K2.6 with Compressed Tensors](https://huggingface.co/moonshotai/Kimi-K2.6)** — A conversational image-text-to-text model from MoonshotAI that leans on compressed tensors for efficient feature extraction. It relies on custom code and is detailed in arXiv:2602.02276, for those who enjoy reading the technical appendix. 📚

## Open Source Releases 📖

- **[claude-code v2.1.118 release](https://github.com/anthropics/claude-code/releases/tag/v2.1.118)** — Adds vim visual and visual-line modes with selection operators, because apparently we needed more ways to exit vim. It also merges `/cost` and `/stats` into a unified `/usage` command and lets you create custom themes via JSON editing. 🛠️
- **[opencode v1.14.21 release](https://github.com/anomalyco/opencode/releases/tag/v1.14.21)** — Now pulls diagnostics from LSP servers for C# and Kotlin, and improves session compaction so your long threads don't lose the plot. It also fixes project detection for bare Git repos and worktrees, a niche but appreciated fix. 🛠️
- **[cline v3.80.0 release](https://github.com/cline/cline/releases/tag/v3.80.0)** — Integrates remote `globalSkills` from enterprise config with UI toggles and updates the onboarding flow to fetch recommended models dynamically. No more hardcoded lists pretending to know what you want. 🛠️
- **[Microsoft Agent Framework 1.1.1 Released](https://pypi.org/project/agent-framework/1.1.1/)** — Microsoft bundles core abstractions and implementations for building AI agents into a single Python package. It includes integrations for OpenAI, Anthropic, Ollama, and Azure AI Search, plus a debug UI with an OpenAI-compatible API server. 📚
- **[ramjetio 0.8.5: Distributed Cache System for PyTorch](https://pypi.org/project/ramjetio/0.8.5/)** — Introduces a distributed cache system aimed at optimizing data handling and reducing I/O bottlenecks during PyTorch model training. If your distributed training jobs are spending more time waiting on data than computing, this might be worth a read. 🔥
- **[SUMD 0.3.33: Structured Markdown for AI Documentation](https://pypi.org/project/sumd/0.3.33/)** — Provides a standardized format for AI-aware project documentation using structured metadata inside Markdown. It’s designed to give AI tooling better context, assuming the tools actually bother to read the docs. ✍️

## AI Dev Tools 📖

- **[goose v1.32.0 release](https://github.com/aaif-goose/goose/releases/tag/v1.32.0)** — Integrates Exa AI-powered search as a tool and adds desktop notifications for when your task finally finishes. It also introduces `@agent` mention support in chat, because why talk to a bot when you can formally address it? 🛠️
